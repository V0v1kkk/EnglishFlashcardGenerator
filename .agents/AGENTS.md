# Privacy & Security Rules

- **NEVER commit the user's real personal notes or data to the git repository.** 
- When generating examples, test data, or output to be committed to git, **ONLY use synthetic data** (e.g., `example/input/sample_notes.md`).
- Ensure directories like `test_run/` or `example/input/` that might contain real data are not committed or tracked by git, unless the data is strictly synthetic.
